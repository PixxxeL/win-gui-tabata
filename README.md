# ScheduleTimer

Настольный таймер-расписание для Windows (WPF, .NET 8). Показывает крупный
циферблат из светящихся рисок, отсчитывает периоды по расписанию из
`config.json`, проигрывает звуки и тиканье часов.

## Возможности

- **Расписания из периодов** с вложенностью в стиле «помодоро»
  (работа → отдых → работа → …).
- **Подготовка + работа**: у каждого периода может быть фаза подготовки
  (обратный отсчёт перед началом) и основная фаза.
- **Повторы** периодов и вложенные периоды, чередующиеся после каждого повтора.
- **Циферблат** из 240 рисок: заполняется цветом периода со свечением
  пропорционально прошедшему времени.
- **Клик по циферблату** «подматывает» время пропорционально углу (перемотка).
- **Звуки** на события периода (старт / предупреждение за 5 c / финиш) —
  форматы WAV и MP3.
- **Тиканье часов** (`zvuk-chasov.mp3`) во время хода; в последние 5 секунд
  периода — громче. Отключается кнопкой 🔊 в заголовке.
- **Перетаскивание окна** за любое место, кроме круга циферблата и кнопок.
- Управление: **Старт/Продолжить**, **Пауза** (повторное нажатие снимает
  с паузы), **Стоп** — мышью или с клавиатуры.

## Горячие клавиши

| Клавиша | Действие                    |
|---------|-----------------------------|
| `Пробел`| Старт / Пауза (тоггл)       |
| `S`     | Стоп                        |
| `M`     | Звук тиканья вкл/выкл        |

Клавиша показывается в подсказке при наведении на кнопку.

## Язык интерфейса

Язык UI выбирается автоматически по языку ОС. Поддерживаются: английский,
русский, китайский, японский, немецкий, испанский, французский, итальянский,
португальский; для остальных — английский (фолбэк). Названия периодов и
расписаний берутся из `config.json` и не переводятся (это ваши данные).

Принудительно задать язык можно переменной окружения, напр.: `SCHEDULETIMER_LANG=ja`.

## Запуск

Звук тиканья и иконка **вшиты в exe**. Рядом с `ScheduleTimer.exe` нужен только
`config.json` (и звуковые файлы периодов, если указали их в конфиге).

## Файл конфигурации `config.json`

Кодировка UTF-8. Корень — объект с массивом расписаний:

```json
{
  "schedules": [
    {
      "name": "Стандартный",
      "active": true,
      "periods": [ /* ... */ ]
    }
  ]
}
```

### Расписание (`schedule`)

| Поле      | Тип       | Описание                                             |
|-----------|-----------|------------------------------------------------------|
| `name`    | строка    | Название (показывается в центре циферблата).         |
| `active`  | bool      | Активное расписание. **Используется только первое** активное. |
| `periods` | массив    | Периоды верхнего уровня.                             |

### Период (`period`)

| Поле       | Тип     | По умолчанию | Описание                                                        |
|------------|---------|--------------|-----------------------------------------------------------------|
| `name`     | строка  | `""`         | Название периода (показывается под временем).                   |
| `color`    | цвет    | `0`          | Цвет активных рисок и свечения. Форматы — ниже.                 |
| `prepare`  | int, c  | `0`          | Подготовка в **секундах** перед стартом. `0` — без подготовки. При `repeat>1` играет **только перед первым повтором**. |
| `duration` | int, c  | `0`          | Длительность основной фазы в **секундах**.                      |
| `repeat`   | int     | `1`          | Сколько раз повторить период.                                   |
| `sound`    | объект  | пусто        | Звуки событий (см. ниже).                                       |
| `periods`  | массив  | `[]`         | Вложенные периоды (помодоро) — вставляются **между** повторами родителя (после последнего повтора не добавляются). |

### Цвет (`color`)

Принимается в любом из форматов:

- число: `10027008`
- hex со `0x`: `"0x990000"`
- hex с `#`: `"#990000"`
- hex без префикса: `"990000"`

Используются младшие 24 бита — `RRGGBB` (альфа игнорируется).

### Звуки (`sound`)

| Поле           | Когда играет                                             |
|----------------|----------------------------------------------------------|
| `start`        | В начале фазы.                                            |
| `beforeFinish` | Когда до конца фазы остаётся 5 секунд.                    |
| `finish`       | В конце фазы.                                             |

Значение — имя файла (`"beep.wav"`, `"gong.mp3"`) рядом с `exe`, либо `null`.
Поддерживаются WAV и MP3.

## Как строится очередь

Подготовка играет один раз (перед первым повтором), затем `repeat` основных фаз,
между которыми вставляются вложенные периоды. Пример для «работа ×3»
с вложенным «отдых» (после последней работы отдыха нет):

```
Работа(подгот.) → Работа → Отдых → Работа → Отдых → Работа
```

Текст под часами по ходу этого примера меняется так:

```
Подготовка → Работа
Работа [1/3] → Отдых
Отдых → Работа
Работа [2/3] → Отдых
Отдых → Работа
Работа [3/3] → Финиш
```

## Примеры

### Минимальный

```json
{
  "schedules": [
    {
      "name": "Помидор 25/5",
      "active": true,
      "periods": [
        {
          "name": "Работа",
          "color": "#C0392B",
          "duration": 1500,
          "repeat": 4,
          "periods": [
            { "name": "Отдых", "color": "#27AE60", "duration": 300 }
          ]
        }
      ]
    }
  ]
}
```
→ 25 минут работы и 5 минут отдыха, 4 круга (после последней работы — тоже отдых).

### С подготовкой и звуками

```json
{
  "schedules": [
    {
      "name": "Тренировка",
      "active": true,
      "periods": [
        {
          "name": "Раунд",
          "color": "0x2980B9",
          "prepare": 10,
          "duration": 180,
          "repeat": 5,
          "sound": {
            "start": "bell.wav",
            "beforeFinish": "beep.wav",
            "finish": "gong.mp3"
          },
          "periods": [
            { "name": "Перерыв", "color": "0x8E44AD", "duration": 60 }
          ]
        }
      ]
    }
  ]
}
```

## Ограничения

- Длительности задаются **целыми секундами**; `prepare`/`duration` — только
  положительные целые.
- Из всех расписаний применяется **только первое** с `active: true`.
- Цвет — 24-битный `RRGGBB`, прозрачность не учитывается.
- Состояние не сохраняется между запусками; «Стоп» перечитывает `config.json`.
- Звуковые файлы и `config.json` должны лежать **рядом с exe**.
- Предупреждающий звук `beforeFinish` срабатывает ровно за 5 секунд до конца,
  поэтому для фаз короче ~6 секунд он может не сработать.

## Сборка

```powershell
dotnet build ScheduleTimer.sln -c Debug
```

Требуется .NET 8 SDK. Зависимость: [NAudio](https://github.com/naudio/NAudio)
(воспроизведение WAV/MP3).
